    hash_sim(hsh1, hsh2) click to toggle source

    Return similarity of 2 hashes using cosine similarity cosine similarity of vector A, B = (A·B)/(|A||B|) Here, 2 vectors must have at least 3 elements

    # File movie_data_P2_reborn.rb, line 60
def hash_sim(hsh1, hsh2)
        common_keys = hsh1.keys & hsh2.keys

        if common_keys.size < 3
                return 0
        else
                sum = 0
                power_sum_hsh1 = 0
                power_sum_hsh2 = 0
                common_keys.each do |k|
                        sum += hsh1[k] * hsh2[k]
                        power_sum_hsh1 += hsh1[k] **2
                        power_sum_hsh2 += hsh2[k] **2
                end
                return (sum.to_f / Math.sqrt(power_sum_hsh1 * power_sum_hsh2) * 100).round(3)
        end
end

def most_similar(u, save = nil)

  user_list = @base_um_hsh.keys
  user_list.delete(u)
  sim_hash = Hash.new
  
  user_list.each do |user|
    sim_hash[user] = similarity(u, user)
  end
  sim_hash_dec = sim_hash.sort_by {|k, v| v}.reverse

  if save == "save"
        sim_file_path = File.dirname(__FILE__) + "/similarity_list"
        f = open(sim_file_path, "w")
         
        sim_hash_dec.each do |k, v|
                f.write("\t#{k}\t\t#{v}%\n")
        end

        puts "Similarity list stored in file #{__FILE__}/similarity_list"
        f.close
  end
  
  return sim_hash_dec
end

    Return a number that indicates the popularity (higher numbers are more popular). Define popularity: Total points / Days between first & last vote If a movie has only one rating, its popularity is 0

    # File movie_data_P2_reborn.rb, line 16
def popularity(movie_id)
  total_rate = 0
  times = []
  days = 0

  @base_mu_hsh[movie_id].each do |u, v|
       total_rate += v[0].to_i
       times.push(v[1].to_i)
  end

  times.sort!
  days = ((times.last - times.first) / (3600 * 24)).to_f
  if days != 0
    return (total_rate.to_f / days).round(2)
  else
    return 0
  end
end

    returns a floating point number between 1.0 and 5.0 as an estimate of what user u would rate movie m. Based on sim_users, calculate their rating on m if they watched it. Prediction of u on m is the mean of ratings of the similar users have on m. Some movies may not be able to give prediction because users may not have seen m.

    # File movie_data_P2_reborn.rb, line 153
def predict(u, m, sim_users)
        ratings = []
        sim_users.each do |user|
                rate = rating(user, m)
                if rate != nil
                        ratings.push(rate.to_i)
                end
        end
        
        prediction = Aux.array_mean(ratings)
        if prediction.nan?
                return 3
        else
                return prediction
        end
        
end

    similarity(user1, user2) click to toggle source

    Generate a number which indicates the similarity in movie preference between user1 and user2 (where larger numbers indicate greater similarity) Using cosine similarity Problem: What if u1 & u2 only have 1 film in common? Solution: Only compare two users with more than 3 movies in common. If less than 3, similarity is 0.

    # File movie_data_P2_reborn.rb, line 82
def similarity(user1, user2)
  user1_hsh = Hash.new
  user2_hsh = Hash.new

  @base_um_hsh[user1].each do |k, v|
       user1_hsh[k] = v[0].to_i
  end
  @base_um_hsh[user2].each do |k, v|
       user2_hsh[k] = v[0].to_i
  end

  return hash_sim(user1_hsh, user2_hsh)
end

def top_sim_user(u)
        sim_users = []
        sim_hash_descend = most_similar(u)
        sim_hash_descend.each do |k, v|
                if v.to_f > 98
                        sim_users.push(k)
                else
                        break
                end
        end
        return sim_users
end
